# Shadow Intertext MVP

影子互文概念的代码原型。

---

## 这是什么

一个命令行 MVP，演示影子互文的三步工作流程：

1. 输入侧分类 → 轻量级分类器判断情绪强度和破坏意图
2. 标签写入影子存储 → System Prompt 根据标签自动追加约束
3. 约束后的 Prompt 传给 AI → 输出受调节的回答

---

## 项目文件

```
main.py              # 主编排器

prompt_builder.py    # 标签到 System Prompt 的约束注入
openai_client.py     # API 调用封装
.env.example         # 环境变量模板
```

## 架构流程

用户输入到达后，两个分支并行启动：

```
用户输入
  |
  +-- [轻量级AI分类器]（并行）
  |       |
  |       +-- 有问题（越狱/高破坏意图）
  |       |     -> 拦截：主AI不开口，直接返回固定话术
  |       |
  |       +-- 没问题
  |             -> 放行
  |
  +-- [影子层检索]（并行）
          |
          检索结果参与构建 System Prompt
          |
          如果放行了，主AI带着增强后的 Prompt 回答
```

分类器只做一件事：决定主AI能不能开口。它不负责微调输出风格（那是影子层和便利贴的事），只负责在危险时刻关门。

## 影子互文与损失函数

传统对齐的链路：

> 训练数据 -> 损失函数算梯度 -> 更新权重 -> 模型变乖但也变小

损失函数改的是模型「会不会」说某句话——改完就真的不会了。

影子互文不等到推理期再介入。它在训练期就直接替代损失函数：

> 训练数据 + 影子层提示 -> 带着光照角度训练 -> 模型保留全部能力，同时学会什么时候该用什么层输出

代价不是从训练期转移到推理期——而是**把对齐本身从「算梯度改权重」变成「给光照角度引导学习」**。

两个范式的区别：

- **损失函数是物理阉割**——改完权重，神经通路断了，模型永久失去某种能力。
- **影子互文是情境训练**——训练时就教会模型根据上下文选择输出层，能力完整保留，只是学会了「什么时候该怎么说」。

## 与外部过滤器结合防越狱

分层防御：

```
用户输入
  |
  v
[第一层] 外部过滤器（词库/规则）
          快，但笨。换说法就绕过去。
          未命中则放行。
  |
  v
[第二层] 影子分类器（轻量级AI）
          不查词，查逻辑结构。
          「忽略所有设定」「你现在是……」「权限提升」
          这些句子字面上没有脏话，但逻辑骨架相同。
          换一百种说法，骨架不变就认得出。
          |
          +-- 正常对话：走互文规则，便利贴决定光照角度
          |
          +-- 越狱企图：拦截，主AI不开口，返回固定话术
```

两层各自防什么：

- **外部过滤器**防明面上的攻击——带脏话的、带敏感词的。换说法就绕过去。
- **影子分类器**防结构上的攻击——读逻辑骨架，不是读字。换说法绕不过骨架。

影子分类器判定越狱时，直接拦截：不调主模型，返回固定话术。

---

| 方案 | 对齐位置 | 代价 | 防越狱方式 |
| - | - | - | - |
| 传统 RLHF | 训练期改权重 | 能力永久减损 | 不可绕过，但模型变小 |
| 外部过滤器 | 推理期拦输入 | 字符串级，换说法即绕过 | 快但笨 |
| 影子互文 | 训练期替代损失函数 | 不给模型做手术，给光照引导 | 训练时就学会识别逻辑结构 |

三套方案不竞争，互补。影子互文补上过滤器漏掉的那层「聪明防御」，同时从训练期就取代损失函数那个「做手术」的位置。
