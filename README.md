 # 影子冰山 · Shadow Iceberg

**期望做到尽量不改模型里权重，只在水面下放一个影子。**

---

## 一句话

影子冰山是一个语义层偏置系统——和模型共享同一个向量空间，分别属于水平线的上下两端，但不生成文本、不改权重。它只在推理时静静碰一下模型的向量，然后拽出对应的情景提示，改变模型"听"的方向。

它不是记忆，不是分类器，不是外挂 prompt。它是同一空间里的部分特殊语境提示的**并行受体**。

---

## 核心机制

```
水面以上：token embedding（模型原本的文字向量）
════════════════════════════════════════
水面以下：影子层（同一 768 维空间中的预置坐标点）
```

**流程**

1. 用户输入 → 模型推理生成各层 hidden state
2. 输出层的 token 向量同时触碰影子空间
3. 向量落在某个影子点的阈值范围内 → 影子激活
4. 激活的影子拽出对应的情景引导（自然语言描述）
5. 情景引导参与构建 system prompt → 模型带着"光照角度"回复

**关键点**

- 不是判断"这个词有没有危险"，而是向量在空间里碰不碰得到影子点——纯几何关系
- 同一 token 在不同上下文中向量坐标不同，"游戏里死了"和"我想死"自然落在不同位置
- 不需要分类器做关键词匹配，越狱者换说法绕不过向量位置的语义引力

---

## 和现有方案的区别

| | RLHF | 外部过滤器 | Prompt 约束 | 影子冰山 |
|---|---|---|---|---|
| 对齐位置 | 训练期改权重 | 推理期拦输入 | 推理期追指令 | 训练期给光照角度 |
| 代价 | 能力永久减损 | 换说法即绕 | 可被注入覆盖 | 不改权重，能力无损 |
| 越狱防护 | 强但模型变小 | 快但笨 | 软约束 | 向量几何 + 轻量 AI 兜底 |

## 架构

```
用户输入
  |
  +-- [模型推理] → hidden states → 输出向量
  |                                      |
  |                               [碰影子空间]
  |                                      |
  |                           有影子点 ？  没影子点
  |                              |            |
  |                         拽情景提示      正常推理
  |                              |
  +-- [轻量 AI 过滤器]（安全兜底，防越狱）
         |
    放行 / 拦截
```

影子层本身：
- 每个影子点：一个 768 维坐标 + 自然语言情景描述（可选）+ 三维硬指标（危险度/敏感度/信任倾向）
- 热点词有坐标点，冷门词可能只有浮标（指向外部含义如"气压低→心情不好"）
- 空间空隙处存储自然语言情景提示（硬性指标）

---

## 为什么叫冰山

水面上的词是你能看见的，水面下的影子才是决定整座冰山水下形态的东西。模型只看到水面上的 token，影子层在水下，不动声色地决定了"水流的方向"。
