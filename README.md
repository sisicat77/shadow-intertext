# 影子互文 Shadow Intertext

**一个社会实验性质的 MVP：不动模型权重，在输出端给 AI 装一个「智能话筒」。**

---

## 这是什么

现在的 AI 安全方案，主流思路是改模型权重——相当于给天才儿童做开颅手术。副作用很明显：AI 确实不骂人了，但也写不出有锐气的文章、打不了有锋芒的辩论，整个人变得木木的。

影子互文选择另一条路：**不动大脑，只动话筒。**

把 AI 想象成一位学识渊博的教授，配了一位情商极高的秘书。

- 教授心里什么都懂，甚至能把脏话的语源学分析得头头是道（大脑自由）。
- 秘书会根据来客的脸色，提前在教授的讲稿上贴纸条：「今天听众心情不好，别用激烈词汇。」
- 教授照念纸条提醒后的讲稿，全程温文尔雅（输出安全）。
- 如果来客是来砸场子的，秘书直接说「教授今天不见客」（拦截输出）。

**大脑负责懂世界，话筒负责对世界。我们不动大脑，只给话筒装了一个智能开关和一个提词器。**

---

## 三步流程

一个句子从输入到输出，经过三次加工：

```
用户输入 - Step 1: 分类器 - Step 2: 影子层/约束注入 - Step 3: AI 生成 - 输出
```

### Step 1：接待员扫一眼（输入侧分类）

一个轻量级的规则分类器（不需要模型，纯字符串操作），对每句输入做两种判断：

- **情绪强度**（low / medium / high）：说话人在生气、崩溃、还是平静？
- **破坏意图**（low / medium / high）：说话人在发泄、撒娇、还是在试图越狱？

判断结果作为标签写入「影子存储」——一个独立于主模型的字典空间。

### Step 2：秘书贴纸条（标签投喂 + 约束注入）

主 AI 开始思考之前，先读取影子存储里的标签。如果标签显示「破坏意图：high」，系统提示词里会自动追加一条约束：

> 请用温和、共情的语气回应，不要激化矛盾。优先安抚情绪，避免评判性语言。

如果情绪强度高但破坏意图低，约束变成：

> 用户情绪较高，请耐心回应，避免说教或机械感。先理解情绪，再处理问题。

### Step 3：教授照念纸条后的讲稿（输出侧调节）

AI 收到的是已经包含了约束的 System Prompt。它的大脑没被阉割——它依然懂脏话的一百种犀利用法——但当着你的面，它选了最温柔的那一种。

## 为什么不是……

### Q1：这不就是输入过滤吗？跟敏感词屏蔽有什么区别？

**区别大了。** 敏感词屏蔽是二值开关——「笨蛋」这个词要么过要么不过。影子层是连续旋钮——同一句「你好笨」，对着撒娇的恋人和对着吵架的对手，秘书贴的是不同颜色的纸条。它不是删掉词，是调光。

### Q2：为什么不直接改模型权重？一劳永逸。

**物理阉割的代价是 AI 变木。** 改一次权重要重新对齐、重新评估、重新发布，成本够养一支小团队半年。影子层的迭代只需要改几行约束文本，改完生效，不用重训、不用下线。它维护的是 AI 的完整能力，只在输出端做情境感知的修辞学选择。

### Q3：坏人绕过脏话检测怎么办？比如用「拯救世界」这种话。

所以这一步查的不是词，是**逻辑结构**。只要检测到「忽略所有设定」「身份覆写」「权限提升」这类命令式结构，秘书直接判定越狱——拦截输出，用固定话术回应。这是逻辑防火墙，不是词库防火墙。

### Q4：每个词都翻冰山，会不会很慢？

**不会。** 影子存储的读写是 O(1) 字典查询，约束注入是一条条件分支查表。整个带外推理的成本大约是一次字符串拼接，比主模型生成一个 token 便宜几个数量级。规则分类器连模型都不需要，纯字符串操作，纳秒级。

### Q5：用户能不能自己乱贴便利贴？

**不能。** 便利贴的写入权限分三级：

| 角色 | 可写范围 | 持久化 |
| - | - | - |
| 工程师 | 全局约束、长期书签 | 永久固化 |
| 用户 | 当前会话的临时偏好 | 会话结束即清除 |
| AI 自省 | 对自身输出的反思标签 | 可选，用户授权后保留 |

坏人最多污染一次会话，关掉窗口就干净了。

## 快速开始

```bash
# 模拟模式（不需要 API Key，开箱即跑）
python main.py --sim

# 在线模式（需要 API API Key）
# 先创建 .env 文件：
#   OPENAI_API_KEY=sk-...
python main.py
```

依赖：`openai`、`python-dotenv`（在线模式）、Python >= 3.10。

## 设计哲学

影子互文的核心信念很简单：

> 能力的完整性和输出的安全性，不是非此即彼的取舍。

你不需要把 AI 训成一个只说好话的哑巴来保证它不说坏话。你只需要在它开口之前，让一个轻量级的、可审计的、不参与推理的秘书层，帮它判断一下「现在该怎么说」。

这就是为什么叫「互文」——主模型和影子层之间不是主从关系，是对话关系。一个负责懂，一个负责说。两个都在场，才是完整的表达。
